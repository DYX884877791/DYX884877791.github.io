# Hadoop运行环境搭建

# 虚拟机环境准备

  1. 虚拟机克隆

  2. 修改静态IP

    • 修改网卡信息(虚拟的Mac地址)：
      [root@hadoop02 ~]$ vim /etc/udev/rules.d/70-persistent-net.rules

      # This file was automatically generated by the /lib/udev/write_net_rules
# program, run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single
# line, and change only the value of the NAME= key.

# PCI device 0x8086:0x100f (e1000)

# PCI device 0x8086:0x100f (e1000)
# 不同机器应该配置不同的Mac地址.
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:0c:29:bb:e4:6e", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"
    • 修改连接的网卡信息：
      [root@hadoop02 ~]$ vim /etc/sysconfig/network-scripts/ifcfg-eth0

      DEVICE=eth0
TYPE=Ethernet
UUID=74d8297d-a658-4f14-909f-fb3c7fdba436
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=static
HWADDR=00:0c:29:bb:e4:6e
DEFROUTE=yes
PEERDNS=yes
PEERROUTES=yes
IPV4_FAILURE_FATAL=yes
IPV6INIT=no
NAME="System eth0"
LAST_CONNECT=1554828622
IPADDR=192.168.13.129
GATEWAY=192.168.13.2
DNS1=192.168.13.2
    • 要修改的地方：
      • ONBOOT=yes（启动机器时自动启动网络）
      • BOOTPROTO=static（手动设定ip地址）
      • HWADDR=00:0c:29:bb:e4:6e（与上面的Mac地址一致）
      • IPADDR=192.168.13.129（设定当前机器的IP地址，设定在与网关同一网段）
      • GATEWAY=192.168.13.2（设定网关与Vmware设定的网络编辑器的网关地址一致）
      • DNS1=192.168.13.2 （设定DNS地址）

  3. 修改主机名（Hostname）

    • 修改当前机器主机名
      [root@hadoop02 ~]$ vim /etc/sysconfig/network

      NETWORKING=yes
HOSTNAME=hadoop02
    • 修改hosts文件
      vim /etc/hosts

      127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
192.168.13.128 hadoop01
192.168.13.129 hadoop02
192.168.13.130 hadoop03
192.168.13.131 hadoop04
    • ping命令查看是否能ping通

  4. 关闭防火墙

    • 停止运行防火墙
      [root@test etc]# service iptables stop
iptables：将链设置为政策 ACCEPT：filter                    [确定]
iptables：清除防火墙规则：                                 [确定]
iptables：正在卸载模块：                                   [确定]
    • 查看防火墙状态
      [root@test etc]# service iptables status
iptables：未运行防火墙。
    • 如果要开启或者重启防火墙运行命令：service iptables start/restart
    • service命令开启以及关闭防火墙为即时生效，下次重启机器的时候会自动复原。
    • 如果要永久关闭防火墙：
      chkconfig iptables off
    • 查看状态
      chkconfig --list iptables
    • 永久启用防火墙
      chkconfig iptables on

  5. 创建用户以及分配权限

    • 创建用户
      [root@test ~]# useradd dyx
    • 分配权限，使用root用户编辑/etc/sudoers文件，sudoers文件默认只有只读权限，如果要修改请添加写权限
      [root@test ~]# cd /etc
[root@test etc]# ll | grep sudoers
-r--r-----.  1 root root   3729 12月  8 2015 sudoers
[root@test etc]# chmod u+x sudoers
[root@test ~]# vim /etc/sudoers 

      ## Allow root to run any commands anywhere
root    ALL=(ALL)       ALL
dyx     ALL=(ALL)       ALL
      添加权限成功!

# 安装JDK以及Hadoop

# 安装JDK

  1. 将jdk-8u144-linux-x64.tar.gz上传至任意目录，方便管理，放置在/opt/software目录下
  2. 将JDK解压至/opt/module目录下
    tar -zxvf jdk-8u144-linux-x64.tar.gz -C /opt/module/
  3. 配置java环境变量 如果当前用户是新建的dyx用户，需要加sudo
    [dyx@hadoop02 module]$ sudo vim /etc/profile
    在/etc/profile文件末尾添加如下内容：
    export JAVA_HOME=/opt/module/jdk1.8.0_144
export PATH=$PATH:$JAVA_HOME/bin
    更新环境变量
    source /etc/profile
