# Java的回调机制

# 同步调用、异步调用与回调

# 同步调用：

最基本并且最简单的一种调用方式，类A的方法a()调用类B的方法b()，一直等待b()方法执行完毕，a()方法继续往下走。这种调用方式适用于方法b()执行时间不长的情况，因为b()方法执行时间一长或者直接阻塞的话，a()方法的余下代码是无法执行下去的，这样会造成整个流程的阻塞。

# 异步调用：

为了解决同步调用可能出现阻塞，导致整个流程卡住而产生的一种调用方式。类A的方法方法a()通过新起线程的方式调用类B的方法b()，代码接着直接往下执行，这样无论方法b()执行时间多久，都不会阻塞住方法a()的执行。但是这种方式，由于方法a()不等待方法b()的执行完成，在方法a()需要方法b()执行结果的情况下（视具体业务而定，有些业务比如启异步线程发个微信通知、刷新一个缓存这种就没必要），必须通过一定的方式对方法b()的执行结果进行监听。在Java中，可以使用Future+Callable的方式做到这一点。

# 回调：

类A的a()方法调用类B的b()方法，在类B的b()方法执行完毕之后主动调用类A的回调方法，这样完成一个回调流程，

# 举栗子：

以老板与员工之间的故事为例，当事情简单时，老板自己就可以去做，当事情变复杂了，就需要分配给员工去做，老板可以做其他事，当员工做完了以后通知老板事情已经做完了，老板拿到事情的完成结果再进行下一步处理，员工做完了以后给老板这件事就相当于“回调”

老板类：

package com.dyx.springboot.callback;

/**
 * 老板类
 */
public class Boss {

   // 老板需要维持一份员工的引用
   private Staff staff;

   // 老板的名字
   private String name;

   // 具体的任务
   private String task;

   public Boss(Staff staff, String task, String name) {
      this.staff = staff;
      this.task = task;
      this.name = name;
   }

   // 老板要做某件事，因为这件事太过复杂，所以交给员工来做
   public void doSomething() {
      staff.handleSomething(this, task);
   }

   // 在回调函数中，老板可以获得任务的完成结果，如果当员工完成了任务，老板决定奖励100块
   public void callBackWithResult(String result) {
      System.out.printf("老板[%s]获得了任务的完成结果>>>>[%s]", name, result);
      if ("完成了".equals(result)) {
         System.out.println("奖励100块!!!");
      }
   }
}

员工类

package com.dyx.springboot.callback;

/**
 * 员工类
 */
public class Staff {

   // 员工来处理事情，当事情处理完后，调用回调函数通知老板，所以该方法中需要维持一份老板的引用
   public void handleSomething(Boss boss, String task) {
      System.out.println("员工: 我开始做事了，具体任务是: " + task);
      try {
         Thread.sleep(20_000L);
      } catch (InterruptedException e) {
         e.printStackTrace();
      }
      System.out.println("员工: 我完成了，调用回调函数通知老板...");
      // 事情处理完后，调用回调方法，通知老板做完了...
      boss.callBackWithResult("完成了");
   }
}

测试类

package com.dyx.springboot.callback;

public class TestCallBack {

   public static void main(String[] args) {
       // 1. 员工
       Staff staff = new Staff();

       // 2. 任务
       String task = "去打扫整层楼的卫生!!!";

       // 3. 老板
       Boss boss = new Boss(staff, task, "李老板");

       // 4. 干任务
       boss.doSomething();
   }
}

这里因为上面的例子只有一个老板，要是多个老板要这个员工来干事怎么办，且不同老板对回调结果的处理不一样，所以定义一个接口，（多个老板抽象为上级），接口中的方法定义了上级对回调结果的处理，不同实现的处理不同

public interface ShangJiDoWork {
   void callBackWithResult(String result);
}

/**
 * 所以员工也需要处理一下，维持一个上级的引用
 */
class Staff_new {
   // 员工来处理事情，当事情处理完后，调用回调函数通知老板，所以该方法中需要维持一份老板的引用
   public void handleSomething(ShangJiDoWork shangJi, String task) {
      System.out.println("员工: 我开始做事了，具体任务是: " + task);
      try {
         Thread.sleep(20_000L);
      } catch (InterruptedException e) {
         e.printStackTrace();
      }
      System.out.println("员工: 我完成了，调用回调函数通知老板...");
      // 事情处理完后，调用回调方法，通知老板做完了...
      shangJi.callBackWithResult(task + "做完了");
   }
}

class Boss1 implements ShangJiDoWork{
   // 老板需要维持一份员工的引用
   private Staff_new staff;

   // 老板的名字
   private String name;
   // 具体的任务
   private String task;
   
   public Boss1(Staff_new staff, String task, String name) {
      this.staff = staff;
      this.task = task;
      this.name = name;
   }

   // 老板要做某件事，因为这件事太过复杂，所以交给员工来做
   public void doSomething() {
      staff.handleSomething(this, task);
   }

   @Override
   public void callBackWithResult(String result) {
      System.out.printf("老板[%s]获得了任务的完成结果>>>>[%s]\n", name, result);
      if ((task + "做完了").equals(result)) {
         System.out.println("奖励500块!!!");
      }
   }
}

class Boss2 implements ShangJiDoWork {
   // 老板需要维持一份员工的引用
   private Staff_new staff;

   // 老板的名字
   private String name;

   // 具体的任务
   private String task;

   public Boss2(Staff_new staff, String task, String name) {
      this.staff = staff;
      this.task = task;
      this.name = name;
   }

   // 老板要做某件事，因为这件事太过复杂，所以交给员工来做
   public void doSomething() {
      staff.handleSomething(this, task);
   }

   @Override
   public void callBackWithResult(String result) {
      System.out.printf("老板[%s]获得了任务的完成结果>>>>[%s]\n", name, result);
      if ((task + "做完了").equals(result)) {
         System.out.println("奖励1000块!!!");
      }
   }
}

class TestCallBack1 {
   public static void main(String[] args) {
      // 1. 员工
      Staff_new staff1 = new Staff_new();
      // 1. 员工
      Staff_new staff2 = new Staff_new();
      // 2. 任务
      String task = "去打扫整层楼的卫生!!!";
      // 2. 老板
      Boss1 boss1 = new Boss1(staff1, task, "王老板");
      boss1.doSomething();
      // 2. 老板
      Boss2 boss2 = new Boss2(staff2, task, "张老板");
      boss2.doSomething();
   }
}

同时调用者可以进行抽象，被调用者也可以进行抽象

上面的例子中类A的a()函数调用类B的b()函数，但是b()函数很耗时，不确定什么时候执行完毕，如果是同步调用的话会等b()执行完成后才往下执行回调类A中的函数，如果是异步回调的话调用了b()函数，虽然b()函数没有执行完,但仍然继续往下执行，为了完成这点，就需要另开一个线程了。
